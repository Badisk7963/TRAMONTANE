# Bolt.new — Integrations Analysis
> Source: support.bolt.new/building/intro-bolt, various integration pages (scraped 2026-03-30)

## Core Integrations
1. **Figma** — design import, convert Figma to UI
2. **GitHub** — version control, backups, collaboration
3. **Expo** — mobile application development
4. **Stripe** — payment handling
5. **Supabase** — database management (alternative to Bolt Cloud DB)
6. **Netlify** — hosting (alternative to Bolt Cloud hosting)
7. **Google SSO** — authentication
8. **Import from Lovable** — competitor migration tool

## MCP Server Support
- Built-in connectors: Notion, Linear, GitHub, others
- Custom MCP server support via URL
- Authentication: API key, MCP OAuth, or None
- Per-project toggle for connectors
- Tool management: enable/disable specific MCP tools
- Auto-enable option for all projects
- Best practice: turn off unused connectors to reduce context/tokens

## Notable Features
- **Import from Lovable** — actively pulling users from competitor
- MCP support is user-facing (not just for the builder)
- Connectors add context to project, increasing token usage
- No API-level integration (unlike Lovable's "any API" support)

## Missing Integrations (vs Lovable)
- No GitLab
- No ElevenLabs
- No Firecrawl
- No Perplexity
- No Shopify
- No Slack
- No Telegram
- No Twilio
- No "any API" generic integration pattern
