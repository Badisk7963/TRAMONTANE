# Lovable — Onboarding Analysis
> Source: docs.lovable.dev/introduction/getting-started, lovable.dev (scraped 2026-03-30)

## First-Time User Flow
1. Land on lovable.dev
2. "Get started" button → signup
3. Describe what you want to build OR select a template
4. Can drop in screenshots and docs as input
5. AI generates working application in real-time
6. Iterate via chat feedback
7. Deploy with one click

## Entry Points
- **Prompt-first:** Type what you want to build
- **Template-first:** Start from template gallery
- **Import:** Screenshots, docs, designs

## Onboarding Friction
- Free plan requires no credit card
- 5 daily credits immediately available
- No complex setup steps visible
- Templates lower the barrier to first output

## Key UX Patterns
- Chat-based interaction throughout
- Real-time preview as AI builds
- Plan mode available from start for thinking before building
- `@file` referencing for precision
- Cross-project referencing within workspace

## Steps to First Output
- ~3 clicks: land → signup → describe → see result
- Very low friction entry
- Real-time streaming shows progress while waiting
