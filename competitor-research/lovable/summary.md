# Lovable — Key Findings Summary

## Strengths
1. **Massive scale:** 36M+ projects, 200K+/day — market leader in adoption
2. **Per-workspace pricing:** $25/mo for unlimited users is aggressive
3. **Plan + Agent mode:** Separating thinking from execution is smart UX
4. **Comprehensive integrations:** 13+ shared connectors, 10+ MCP servers, any-API support
5. **Strong security posture:** SOC 2 Type II, ISO 27001, GDPR, data residency in EU/US/AU
6. **No training on customer data** — contractually guaranteed
7. **Real-time streaming** during generation
8. **Cross-project referencing** within workspaces
9. **Student discount** (50% off) — growth play
10. **Lovable 2.0:** Multiplayer collaboration

## Weaknesses
1. **Credit system is opaque** — unclear cost per action
2. **No Mistral models** — all Gemini/GPT, no EU-native AI
3. **No model routing intelligence** — user picks or gets default
4. **No cost transparency** per-call or budget controls
5. **DPA only on Business+ plans** — not available on Pro
6. **US primary entity** (Delaware) despite Swedish roots
7. **JavaScript/React focused** — limited backend flexibility
8. **No self-hosting option** mentioned
9. **No mobile app support** (web-only)
10. **Service Data clause** allows de-identified training

## What ArkhosAI Can Learn
- Plan + Agent mode separation is a proven UX pattern — adopt it
- Per-workspace pricing beats per-user for team adoption
- Template gallery drives discovery and reduces first-use friction
- Community "Discover" section (apps built by users) is good social proof
- Integration gateway handling OAuth automatically is a great DX
- Student discount is smart growth strategy
- Security scanning built into development workflow is table stakes

## ArkhosAI Differentiators vs Lovable
- **Mistral-native** — EU AI models, no dependency on US AI providers
- **Transparent cost routing** — show cost per call, budget controls
- **Model routing intelligence** — automatic model selection based on task
- **GDPR-first** — EU entity, EU infrastructure, no data leaves EU
- **Self-hostable** — open-core model vs closed platform
- **Multi-model transparency** — users see which model, why, and cost
